# Lighthouse 모니터링 시스템 — 화면 및 백엔드 요구사항 정의서

---

## 1. 화면 요구사항 (Frontend)

### 1.1 공통

- 메뉴 구조: **대시보드(Overview)** / **로그 조회(Logs)** 2개 메뉴
- 상단 글로벌 필터: 서비스 선택 (현재는 단일 서비스이나 확장 대비)
- 타임존: 모든 시간 표시는 KST(Asia/Seoul) 기준
- 반응형 레이아웃: 최소 1280px 이상 지원

---

### 1.2 대시보드 (Overview)

#### 1.2.1 필터 영역

| 항목 | 설명 |
|------|------|
| 기간 선택 | 시작~끝 날짜시간 DateTimePicker |
| 기간 프리셋 | 최근 15분 / 1시간 / 6시간 / 24시간 / 7일 버튼 |
| 시간 간격 | 10분 / 30분 / 1시간 선택 (기본값: 1시간) |

- 프리셋 클릭 시 기간 선택기가 자동으로 해당 범위로 세팅
- 간격 선택 변경 시 차트 자동 갱신

#### 1.2.2 요약 카드

| 카드 | 값 | 설명 |
|------|----|------|
| 총 요청 수 | 정수 | 선택 기간 내 전체 요청 건수 |
| 에러 건수 | 정수 | HTTP 상태 코드 4xx + 5xx 건수 |
| 평균 응답 시간 | ms | 선택 기간 내 전체 요청의 평균 response_time_ms |

- 이전 동일 기간 대비 증감률 표시 (선택사항, 2차)

#### 1.2.3 시계열 차트 — 시간대별 요청량

| 항목 | 설명 |
|------|------|
| 차트 유형 | Bar Chart (막대 그래프) |
| X축 | 시간 구간 (선택한 간격 기준) |
| Y축 | 요청 건수 |
| 인터랙션 | 막대 hover 시 해당 구간 요청 수 툴팁 표시 |
| 드래그 선택 | 구간 드래그 시 해당 시간 범위를 파라미터로 로그 조회 페이지로 이동 |

#### 1.2.4 시계열 차트 — 시간대별 응답 시간

| 항목 | 설명 |
|------|------|
| 차트 유형 | Line Chart (라인 그래프) |
| X축 | 시간 구간 (선택한 간격 기준) |
| Y축 | 응답 시간 (ms) |
| 라인 | P95 (주황), P99 (빨강) 두 개 라인 |
| 인터랙션 | hover 시 해당 구간 P95/P99 값 툴팁 표시 |

#### 1.2.5 테이블 — 응답 느린 TOP API

| 컬럼 | 설명 |
|------|------|
| 순위 | 1~10 |
| Method | HTTP 메서드 (GET/POST/PUT/DELETE 등), 메서드별 색상 배지 |
| Endpoint | API 경로 |
| P95 | P95 응답 시간 |
| Avg | 평균 응답 시간 |
| Requests | 요청 건수 |

- P95 내림차순 정렬
- 상위 10개 표시
- 행 클릭 시 해당 엔드포인트 + 선택 기간으로 로그 조회 페이지 이동

#### 1.2.6 테이블 — 최근 에러 로그

| 컬럼 | 설명 |
|------|------|
| 시간 | 타임스탬프 (KST) |
| Level | ERROR / WARN 배지 |
| Method | HTTP 메서드 |
| Endpoint | API 경로 |
| Status | HTTP 상태 코드 |
| Exception | exception_class (패키지 제외 클래스명만 표시) |

- 최신순 정렬, 최근 20건 표시
- 행 클릭 시 로그 조회 페이지로 이동 (해당 로그 상세 패널 열린 상태)

### 1.3 로그 조회 (Logs)

#### 1.3.1 필터 영역

| 필터 | 타입 | 설명 |
|------|------|------|
| 기간 선택 | DateTimePicker | 시작~끝 (필수, 미입력 시 최근 1시간) |
| 기간 프리셋 | 버튼 | 최근 15분 / 1시간 / 6시간 / 24시간 / 7일 |
| 로그 레벨 | 멀티 선택 | ERROR / WARN / INFO (기본: 전체 선택) |
| HTTP 상태 코드 | 멀티 선택 | 2xx / 4xx / 5xx 그룹 또는 개별 코드 |
| HTTP 메서드 | 멀티 선택 | GET / POST / PUT / DELETE / PATCH / OPTIONS |
| 엔드포인트 | 텍스트 입력 | 부분 일치 검색 (예: /api/orders 입력 시 /api/orders/123도 매칭) |
| 응답 시간 | 범위 입력 | 최소~최대 ms (예: 1000ms 이상만 조회) |

- 검색 버튼 클릭 또는 Enter 시 조회 실행
- 필터 초기화 버튼

#### 1.3.2 URL 상태 동기화

- 모든 필터 조건이 URL 쿼리 파라미터에 반영
- 예: /logs?from=2026-03-06T15:00&to=2026-03-06T16:00&level=ERROR,WARN&status=5xx
- 대시보드에서 넘어올 때 파라미터가 자동으로 필터에 세팅
- URL 공유 및 북마크 가능

#### 1.3.3 로그 테이블

| 컬럼 | 설명 | 정렬 |
|------|------|------|
| 시간 | 타임스탬프 (KST, ms 단위까지 표시) | 기본 최신순 |
| Level | INFO / WARN / ERROR 배지 (색상 구분) | 가능 |
| Method | HTTP 메서드 배지 | 가능 |
| Endpoint | API 경로 | 가능 |
| Status | HTTP 상태 코드 (색상: 2xx 초록, 4xx 노랑, 5xx 빨강) | 가능 |
| 응답 시간 | response_time_ms (단위 표시) | 가능 |
| Client IP | 요청자 IP | - |

- 행 전체 색상: ERROR 행은 연한 빨강 배경, WARN 행은 연한 노랑 배경
- 페이지네이션: 페이지당 50건, 페이지 이동 및 총 건수 표시
- 전체 컬럼 정렬 지원 (클릭 시 ASC/DESC 토글)

#### 1.3.4 로그 상세 패널

- 테이블 행 클릭 시 해당 행 아래 또는 오른쪽에 상세 패널 확장
- 상세 패널 표시 내용:

| 필드 | 설명 |
|------|------|
| @timestamp | 전체 타임스탬프 |
| service | 서비스명 |
| host | 호스트명 |
| level | 로그 레벨 |
| http_method | HTTP 메서드 |
| http_path | 엔드포인트 |
| http_status | 상태 코드 |
| response_time_ms | 응답 시간 |
| client_ip | 클라이언트 IP |
| thread_name | 스레드명 |
| exception_class | 예외 클래스 전체명 |
| stack_trace | 스택트레이스 (코드 블록 포맷, 접기/펼치기 지원) |
| message | 원본 메시지 |

- exception_class 클릭 시 동일 예외로 필터링하는 빠른 필터 링크
- 상세 패널에서 JSON 원본 보기 토글

#### 1.3.5 대시보드 연동

- 대시보드 요청량 차트에서 구간 드래그 → 해당 시간 범위로 로그 페이지 이동
- 대시보드 느린 TOP API 테이블 행 클릭 → 해당 엔드포인트 + 시간 범위로 로그 페이지 이동
- 대시보드 에러 로그 테이블 행 클릭 → 해당 로그의 상세 패널이 열린 상태로 로그 페이지 이동
